2023 – DoorGuardian: Autonomous Security System
Contributors: Nippun Sabharwal, Vayun Gupta, Siddarth Natarajan
An open-source, low-power remote door-unlock prototype that replaces ID-cards with on-device face recognition and a chat-based interface.
  • Challenge: Campus and home access rely on plastic cards or keys, creating delays, extra costs and accessibility hurdles.
  • Solution: An ultrasonic sensor wakes an Arduino UNO and OV7670 camera when someone approaches. The image streams via UART → PC → Telegram bot. The owner sends “door open”/“door close” over Telegram, and an ESP32 toggles SG90 servos to lock or unlock.
  • Tech Stack:
    • Sensing & Power: HC-SR04 ultrasonic for 50 cm trigger & power-saving standby
    • Imaging: OV7670 camera module over UART
    • Control: Arduino UNO (coordination), ESP32 (Wi-Fi, Telegram API)
    • Actuation: SG90 servo motors as the lock mechanism
    • Integration: Telegram Bot via Make.com for workflow automation
  • Key Features: live video feed playback, automated entry/exit logs, easy face-database updates, upgradable firmware
  • Real-World Use Cases: Airbnb hosts granting secure, keyless guest access; campus buildings offering ID-free entry; improved accessibility for users with disabilities
  • Repository: remote-unlock
2024 – Fully Functional 16-bit CPU Core & Verilog Autonomous Agent
GitHub: svcpu
  • SLC-3 ISA in SystemVerilog: Implemented a 16-bit processor with 16-bit PC, IR, registers, ALU and a 3-stage fetch–decode–execute pipeline.
  • Control Unit FSM: Built a state-machine sequencing memory access, ALU ops and I/O interactions via a `cpu_to_io` bridge.
  • Memory-Mapped I/O & BRAM: Handled peripherals using on-chip Spartan-7 block RAM without an external "ready" signal.
  • FPGA-Verified: Synthesized and verified on Spartan-7 board, operating stably at 50 MHz.
  • Verilog Autonomous Agent: LLM-based system that generates, synthesizes, and verifies new RTL modules, reducing design iteration by 40%.
2025 – Full RISC-V OS Kernel & KTFS Journaling Filesystem + Drivers
GitHub: ktfs
  • UNIX-style Kernel in C:
    • Bootloader, trap handling, Sv39 paging, process abstraction
    • User-mode syscalls: open/close/read/write/ioctl/exec/fork/wait/usleep/fscreate/fsdelete
  • KTFS Filesystem: Block-based with journaling, metadata crash recovery, supports indirection and multiple mount types.
  • Drivers: UART, RTC, PLIC, VirtIO block & RNG, GPIO, SPI, servo control.
  • Block Cache: Write-back with journaling consistency, configurable associativity.
  • Threading: Preemptive/cooperative with mtime timer, context switching, idle thread.
  • ELF Loader: Runs user ELF binaries like Star Trek game via I/O interface.
  • Validation: Unit tests, gdb debugging, full integration on QEMU RISC-V.
2024 – VR-Based Robot Teleoperation Interface
  • Head-driven control: Captured 6-DOF head-pose from Oculus at 60 Hz over ROS; mapped to robotic neck joints via imitation learning.
  • Immersive 3D feedback: ZED Mini stereo RGBD camera into Unity, sub-60 ms latency stereo feed.
  • Latency & smoothing: ROS filters + H.264/H.265 compression, achieving 45–60 ms round-trip delay.
  • Modular pipeline: Unity–ROS–Dynamixel setup, extensible to other robot configurations.
  • Demonstrated performance: ±0.5° joint accuracy across 3,000-mile network for folding and assembly tasks.
